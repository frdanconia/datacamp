The importance of flat files in data science¶

Pop quiz: what exactly are flat files?¶

  • Flat files consist of rows and each row is called a record.
  • A record in a flat file is composed of fields or attributes, each of which contains at most one item of information.
  • Flat files are pervasive in data science.

Why we like flat files and the Zen of Python¶

  • Guido van Rossum wrote python and PEP8 is his style guide
  • The zen of python suggest that 'flat is better than nested'

Working with mixed datatypes (1)¶

  • Here, the first argument is the filename, the second specifies the delimiter , and the third argument names tells us there is a header.
  • Because the data are of different types, data is an object called a structured array.
  • Because numpy arrays have to contain elements that are all the same type, the structured array solves this by being a 1D array, where each element of the array is a row of the flat file imported.

What a data scientist really needs:¶

  • Two-dimensional labeld dsata structures
  • Columns of potentially different types
  • Manipulate, slice, reshape, groupby, join, merge
  • perform statistics easily
  • work with time series data

Pandas and the Dataframe¶

  • Wes McKiney created pandas
  • The documentation for the library is great
  • "Python has long been great for data munging and preparation, but less so for data analysis and modeling. Pandas helpls fill this gap, enabling you to carry out your entire data anaysis workflow in Python without having to switch to a more domain specific language like R.' :)
  • Python dataframe is the pythonic analog of R's data frame
  • "A matrix has rows and columns, a dataframe has observations and variables" - Hadley Wickham

Working with relational databases in Python¶

Introduction to relational databases¶

Pop quiz: The relational model¶

  • Each row or record in a table represents an instance of an entity type.
  • Each column in a table represents an attribute or feature of an instance.
  • Every table contains a primary key column, which has a unique entry for each row.
  • There are relations between tables.

Creating a database engine in Python¶

Creating a database engine¶

  • SQLite database
    • Fast and simple
  • SQLAlchemy
    • Works with many relational database management systems

Creating a database engine¶

  • Here, 'sqlite:///Chinook.sqlite' is called the connection string to the SQLite database Chinook.sqlite.
